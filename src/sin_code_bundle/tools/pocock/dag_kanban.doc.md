# dag_kanban.py - DAG Task Parser & Orchestrator

## Purpose

Parses the PRD.md generated by `grill_me.py` and builds a Directed Acyclic Graph (DAG)
of tasks, identifying dependencies and computing optimal execution order for parallel execution.

## What it does

1. **PRD Parsing**: Extracts task slices from PRD.md (Tracer Bullets, Slices, Tasks)
2. **DAG Construction**: Builds adjacency list and in-degree map
3. **Topological Sort**: Kahn's algorithm for optimal execution order
4. **Parallel Grouping**: Identifies tasks that can run simultaneously
5. **Docker Export**: Generates docker-compose for containerized execution

## Dependencies

- `collections` - defaultdict, deque
- `re` - Regex for PRD parsing
- `json` - JSON export
- `yaml` - Docker Compose generation (optional)
- `argparse` - CLI interface

## Usage

```bash
# Parse PRD.md and show execution order
python3 -m sin_code_bundle.tools.pocock.dag_kanban

# Specify different PRD
python3 -m sin_code_bundle.tools.pocock.dag_kanban --prd docs/PRD.md

# Export JSON
python3 -m sin_code_bundle.tools.pocock.dag_kanban --json

# Generate Docker Compose
python3 -m sin_code_bundle.tools.pocock.dag_kanban --docker --output docker-compose.yml
```

## PRD Format Expected

```markdown
## Technische Spezifikation
### Vertikale Architekturschnitte (Tracer Bullets)
- [ ] Slice 1: Datenbankschema & API-Schnittstelle
- [ ] Slice 2: Integrations-Route & Validierungs-Logik
- [ ] Slice 3: Frontend-Hook / CLI-Integration
```

## Output Format

```json
{
  "tasks": {
    "slice_1": {
      "id": "slice_1",
      "label": "Slice 1",
      "description": "Datenbankschema & API-Schnittstelle",
      "dependencies": [],
      "status": "pending",
      "executor": "agent-1",
      "container": "container-slice_1"
    }
  },
  "execution_order": ["slice_1", "slice_2", "slice_3"],
  "parallel_groups": [
    ["slice_1"],
    ["slice_2"],
    ["slice_3"]
  ]
}
```

## Integration with Workflow

1. **Consumes**: `PRD.md` from `grill_me.py`
2. **Produces**: Task order, parallel groups, Docker Compose
3. **Feeds**: `teammate-adapter.js` for task distribution
4. **Monitors**: `opencode-cleanup-hook.sh` for stale locks

## Key Features

- **Automatic dependency detection** - Sequential tasks depend on previous
- **Kahn's algorithm** - O(V+E) topological sort
- **Parallel group detection** - Identifies tasks for concurrent execution
- **Docker Compose export** - Ready for containerized orchestration
- **JSON export** - Programmatic consumption
- **Circular dependency detection** - Fails fast on invalid DAGs

## Known Caveats

- Requires PRD in specific format (Markdown task list)
- Manual dependencies not yet supported (only sequential)
- Docker Compose requires YAML package
- Assumes linear task flow unless specified
